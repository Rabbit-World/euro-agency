1. Project Overview

Game Concept and Educational Purpose

The GDPR Knowledge Challenge is an interactive educational game designed to help users learn about the General Data Protection Regulation (GDPR) in an engaging way. The game presents players with quiz questions covering key GDPR concepts, principles, rights, and compliance requirements, making complex regulatory information more accessible and memorable.

The educational purpose of the game is threefold:

  • To increase awareness and understanding of GDPR principles and requirements
  • To help users recognize their rights as data subjects under GDPR
  • To demonstrate GDPR compliance in practice through the game's own privacy features

Target Audience and Learning Objectives

The game targets a diverse audience including:

  • Professionals who need to understand GDPR for their work
  • Students learning about data protection and privacy regulations
  • General public interested in understanding their data privacy rights
  • Organizations seeking to train their staff on GDPR compliance

Key learning objectives include:

  • Understanding the scope and applicability of GDPR
  • Recognizing the seven fundamental principles of data handling
  • Identifying key rights of data subjects
  • Learning about organizational responsibilities and compliance requirements
  • Understanding data breach notification requirements
  • Recognizing potential penalties for non-compliance

Key Features Implemented

Interactive Quiz System

Multiple-choice questions with three difficulty levels (easy, medium, hard) and explanations for each answer.

Country Selection

Players can select their country from all 27 EU member states or a non-EU option, affecting leaderboard categorization.

Dual Leaderboards

Separate EU and global leaderboards with visual charts and rankings to track performance.

GDPR-Compliant Consent System

Comprehensive consent management with granular options for necessary, preferences, and analytics data.

Responsive Design

Fully responsive interface that works across desktop, tablet, and mobile devices.

Accessibility Features

WCAG 2.1 AA compliant with screen reader support, keyboard navigation, and high contrast mode.

2. Technical Implementation

Architecture Overview

The GDPR Knowledge Challenge follows a modular architecture with clear separation of concerns. The application is built using vanilla JavaScript with a component-based approach, making it lightweight and performant without requiring external frameworks.

Technologies Used

The game is built using modern web technologies without external dependencies:

  • HTML5: Semantic markup for structure and accessibility
  • CSS3: Custom styling with CSS variables, flexbox, and grid for responsive layouts
  • JavaScript: Vanilla JS for all game logic and interactivity
  • LocalStorage API: For client-side data persistence with proper consent
  • SVG: For interactive chart visualizations in the leaderboards

3. Game Features

Quiz System with Difficulty Levels

The game features a comprehensive quiz system with 15 questions about GDPR, organized into three difficulty levels:

  • Easy (5 questions): Covering basic GDPR definitions, scope, and fundamental concepts
  • Medium (5 questions): Focusing on individual rights, data breach notifications, and core principles
  • Hard (5 questions): Addressing detailed compliance requirements, technical implementations, and special categories of data

Players can choose their preferred difficulty level or select "Mixed" to get questions from all difficulty levels. Each question includes:

  • Clear question text
  • Four multiple-choice options
  • Detailed explanation after answering
  • 30-second timer to add challenge

  • EU Leaderboard: Shows rankings of players from EU member states
  • Global Leaderboard: Shows rankings of all players regardless of location

Both leaderboards include:

  • Visual chart representation of scores
  • Ranking position with country code
  • Score display with percentage
  • Rank change indicators (up, down, same, or new)
  • Filtering options by difficulty level

Ranking System and Visualizations

The game features a sophisticated ranking system that calculates scores based on multiple factors:

  • Base Score: Percentage of correct answers
  • Time Bonus: Additional points for faster completion
  • Difficulty Multiplier: Higher multipliers for more difficult levels (Easy: 1x, Medium: 1.25x, Hard: 1.5x, Mixed: 1.2x)

Players are assigned to performance tiers based on their scores:

Tier Score Threshold Color Code
Expert 90% and above Dark Green
Advanced 70% - 89% Green
Intermediate 50% - 69% Yellow
Beginner Below 50% Red

The leaderboard visualizations include:

  • SVG-based bar charts showing relative performance
  • Color-coded bars based on performance tier
  • Highlighting for the current player's position
  • Animations for rank changes and new entries
  • Accessible design with ARIA attributes and screen reader support

GDPR-Compliant Privacy Features

As an educational game about GDPR, the application implements comprehensive privacy features that serve both as functional components and educational examples:

  • Granular Consent Management:
    • Clear consent banner on first visit
    • Three consent categories: necessary, preferences, and analytics
    • Option to accept all, accept only necessary, or customize settings
    • Ability to withdraw or modify consent at any time
  • Transparent Data Collection:
    • Clear explanation of what data is collected and why
    • No collection of special categories of personal data
    • Data minimization principle applied throughout
  • User Rights Implementation:
    • "Download My Data" functionality
    • "Delete My Data" option
    • Comprehensive privacy policy with rights explanation
  • Data Storage Limitations:
    • Client-side storage only (no server storage in current implementation)
    • Leaderboard data retained for maximum of 12 months
    • Option to play without storing any personal data

The visual design of the GDPR Knowledge Challenge follows a clean, professional aesthetic that balances educational seriousness with engaging gameplay elements:

  • Color Palette:
    • Primary Color: #2c6bac (EU Blue) - Representing the European Union theme
    • Secondary Color: #34a853 (Green) - For success states and positive feedback
    • Accent Color: #f5b400 (Yellow/Gold) - For highlights and calls to action
    • Error Color: #d93025 (Red) - For incorrect answers and warnings
    • Neutral backgrounds and text colors for readability
  • Typography:
    • System font stack for optimal performance and native feel
    • Base font size of 16px with relative units for accessibility
    • Clear typographic hierarchy with distinct heading styles
    • Adequate line height (1.5) for improved readability
  • Layout Principles:
    • Focused, single-purpose screens with clear progression
    • Consistent spacing and alignment using CSS variables
    • Visual separation between different functional areas
    • Prominent calls to action with clear visual hierarchy
  • Interactive Elements:
    • Clear hover and focus states for all interactive elements
    • Visual feedback for user actions (selections, submissions)
    • Animated transitions between states for better user experience
    • Consistent button styling with purpose-specific colors

Responsive Design Implementation

The game is fully responsive, adapting to different screen sizes and devices:

Desktop (>768px)

  • Two-column layout for welcome screen
  • Side-by-side leaderboards
  • Spacious quiz interface with large answer options
  • Horizontal navigation and consent controls
  • Detailed chart visualizations

Mobile (<768px)

  • Single-column stacked layout
  • Vertically stacked leaderboards
  • Optimized touch targets (min 44px)
  • Vertical navigation and consent controls
  • Simplified chart visualizations
  • Adjusted spacing and typography

The responsive implementation uses:

  • CSS Flexbox and Grid for flexible layouts
  • Media queries for breakpoint-specific styles
  • Relative units (rem, %) for scalable components
  • Viewport meta tag for proper mobile rendering
  • Touch-friendly interaction targets

Accessibility Features (WCAG 2.1 AA Compliance)

The game implements comprehensive accessibility features to ensure it's usable by people with various disabilities:

  • Keyboard Navigation:
    • All interactive elements are keyboard accessible
    • Logical tab order follows visual layout
    • Focus indicators are clearly visible
    • Skip link for keyboard users to bypass navigation
  • Screen Reader Support:
    • Semantic HTML structure with appropriate landmarks
    • ARIA attributes for dynamic content
    • Descriptive alt text for images
    • Announcements for score updates and feedback
  • Visual Accessibility:
    • Color contrast ratios meeting WCAG AA standards (minimum 4.5:1 for normal text)
    • Color is not the only means of conveying information
    • Text can be resized up to 200% without loss of functionality
    • High contrast mode support via media queries
  • Cognitive Accessibility:
    • Clear, consistent navigation
    • Simple language for instructions
    • Error prevention and clear error messages
    • Reduced motion option for animations
